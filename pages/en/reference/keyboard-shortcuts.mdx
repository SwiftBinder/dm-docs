# Keyboard Shortcuts

Complete reference for keyboard shortcuts and the command palette in Document Manager.

---

## Overview

Document Manager supports keyboard shortcuts for faster navigation and actions. Using shortcuts can significantly speed up your workflow.

---

## Command Palette

The command palette is the most powerful keyboard feature — access anything with a few keystrokes.

### Opening the Command Palette

| Platform | Shortcut |
|----------|----------|
| **Windows/Linux** | `Ctrl + K` |
| **macOS** | `Cmd + K` |

### Using the Command Palette

1. Press `Ctrl+K` (or `Cmd+K`)
2. Start typing:
   - Document name
   - Page name
   - Action
3. Use arrow keys to navigate results
4. Press `Enter` to select

### What You Can Do

| Type | Result |
|------|--------|
| Document name | Jump to that document |
| "Documents" | Go to Documents page |
| "Projects" | Go to Projects page |
| "Settings" | Open Settings |
| "Home" | Go to Dashboard |
| Project name | Navigate to project |

### Command Palette Tips

- Start typing immediately — no need to click
- Partial matches work ("rep" finds "Report")
- Most recent items appear first
- Press `Esc` to close

## Navigation Shortcuts

### Go To Shortcuts

Use these from anywhere to quickly navigate:

| Shortcut | Destination |
|----------|-------------|
| `G` then `H` | Home (Dashboard) |
| `G` then `D` | Documents |
| `G` then `P` | Projects |
| `G` then `T` | Teams |
| `G` then `S` | Settings |

*Press the keys in sequence, not simultaneously*

### How Sequential Shortcuts Work

1. Press `G` (for "Go to")
2. Within 1 second, press the second key
3. You'll navigate to that page

## Document List Shortcuts

When viewing the Documents list:

| Shortcut | Action |
|----------|--------|
| `J` | Move to next document |
| `K` | Move to previous document |
| `Enter` | Open selected document |
| `/` | Focus search box |
| `Esc` | Clear search / deselect |

### Selection Navigation

| Shortcut | Action |
|----------|--------|
| `↓` or `J` | Move selection down |
| `↑` or `K` | Move selection up |
| `Enter` | Open selected item |

When viewing a document:

| Shortcut | Action |
|----------|--------|

| `D` | Download document |
| `C` | Go to Comments tab |
| `V` | Go to Versions tab |

## Form Shortcuts

When editing or in dialogs:

| Shortcut | Action |
|----------|--------|
| `Ctrl+S` / `Cmd+S` | Save |
| `Esc` | Cancel / Close |
| `Tab` | Next field |
| `Shift+Tab` | Previous field |
| `Enter` | Submit form (in some contexts) |

## Table Navigation

When in table views:

| Shortcut | Action |
|----------|--------|

| `↑` | Previous row |

- All interactive elements have ARIA labels
- Focus management for dialogs

- Focused elements are highlighted
- Tab order follows logical sequence
- Skip links available for navigation

- Animations respect `prefers-reduced-motion`
- Essential transitions are minimal

## Tips for Power Users

### Efficiency Patterns

1. **Command Palette First**
   - For most navigation, `Ctrl+K` is fastest
   - Type a few letters of destination
   - Press Enter

2. **Learn Go-To Shortcuts**
   - `G` then `H` → Home
   - `G` then `D` → Documents
   - Muscle memory builds quickly

3. **Keyboard-First Editing**
   - Tab through fields
   - `Ctrl+S` to save
   - `Esc` to cancel

4. **List Navigation**
   - Use `J`/`K` or arrows
   - `Enter` to open
   - `/` to search

### Daily Workflow Example

```
1. Press Ctrl+K → type document name → Enter
   (Opens the document directly)

2. Press V → view versions

3. Press C → add comment

4. Press G then D → back to Documents list

5. Press / → search for next document
```

## Troubleshooting

### Shortcuts Not Working

**Check:**
- Focus is in the main window (not a text field)
- No dialog is open
- Browser didn't capture the shortcut
- JavaScript is enabled

### Command Palette Empty

**Check:**
- Type something to search
- Wait a moment for results
- Try refreshing the page

### Navigation Shortcuts Failing

**Try:**
- Press keys more slowly (for sequential shortcuts)
- Make sure you're not in an input field
- Check keyboard layout

### Conflict with Browser

If a shortcut conflicts with your browser:
- Use an alternative shortcut if available
- Check browser settings for customization
- Some browser shortcuts take precedence
